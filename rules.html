Matching Rules Documentation

How MIMIC-IV demo patients are evaluated against clinical trial eligibility criteria · Multi-Trial View · Single Trial Screening

1. Overview

Each clinical trial has eligibility criteria written as free text (e.g., "Age ≥ 50 years", "No history of heart failure"). Our system evaluates each criterion against each patient's structured EHR data from the MIMIC-IV demo dataset using a two-step process:

  1. Classify the criterion by EHR skill level (S0–S3)
  2. Evaluate using regex-based extraction + patient data lookup

The rules are heuristic pattern matching, not validated clinical screening logic. They demonstrate how structured EHR data can be used for automated pre-screening, with known limitations.

2. S0–S3 Skill Classification

Every criterion is classified into one of four skill levels, based on what data source would be needed to evaluate it. Classification uses regex pattern matching on the criterion text.

SkillNameDescriptionCan We Evaluate?Example Patterns Matched
S0 Undeterminable Cannot be assessed from EHR — requires prospective data, consent, or clinical judgment No informed consent, willing to comply, life expectancy, investigator's judgment, planning to
S1 Structured Data Directly evaluable via SQL on EHR structured tables (labs, vitals, demographics, prescriptions) Yes age ≥ 18, creatinine ≤ 2.0, hemoglobin, currently taking aspirin, female patients
S2 Clinical Notes Requires NLP/LLM search in unstructured clinical notes (history, subjective status) No* history of smoking, functional status, NYHA class, cognitive impairment, prior surgery
S3 Hybrid Try structured EHR data first (ICD codes); fall back to clinical notes if unknown Partial heart failure, CABG, diabetes mellitus, atrial fibrillation, chronic kidney disease

* S2 criteria cannot be evaluated because the MIMIC-IV demo dataset does not include clinical notes. The full MIMIC-IV-Note module would be needed.

Classification Priority

When a criterion matches multiple skill patterns, the first match wins in this order:

S0 → S1 → S3 → S2 (default)S0 is checked first because consent/admin criteria override other keywords. S2 is the fallback if nothing else matches.

3. Evaluator Rules

For criteria classified as evaluable (mainly S1 and S3), the system tries five evaluators in order. The first match wins — once an evaluator returns a result, the others are skipped.

evalAge → evalSex → evalLab → evalMedication → evalDiagnosisIf none match, the criterion is marked as "Cannot evaluate from structured EHR data"

3a. Age Evaluator

Extracts age constraints from text using 7 regex patterns, then compares against the patient's anchor_age from MIMIC-IV patients.csv.

Pattern TypeExample TextWhat It Extracts
Minimum"≥18 years", "at least 50 years", "older than 21"age >= value
Maximum"≤75 years", "younger than 80", "up to 65"age <= value
Range"age 18-75 years", "aged 50 to 80"value1 <= age <= value2
Criterion: "Age ≥ 50 years" → extracts min=50 → patient age 72 → PASS "Age 72 >= 50"

3d. Medication Evaluator

Checks if patients are taking (or not taking) specific drugs. Uses two approaches:

Individual drugs — 32 drug names searched in criterion text and patient's prescriptions.csv:

aspirin, warfarin, clopidogrel, heparin, enoxaparin, insulin, metformin, atorvastatin, rosuvastatin, simvastatin, lisinopril, enalapril, ramipril, losartan, valsartan, metoprolol, carvedilol, atenolol, bisoprolol, amlodipine, amiodarone, digoxin, furosemide, spironolactone, hydrochlorothiazide, rivaroxaban, apixaban, dabigatran, ticagrelor, prasugrel, nitroglycerin

Drug classes — 6 class patterns mapped to specific drugs:

Negation detection: If the criterion text contains words like "not", "no", "without", "never", "exclude", "prohibited", the logic is inverted — having the drug means FAIL.

Criterion (inclusion): "Currently on stable statin therapy" → checks for statin class → patient has atorvastatin → PASS
Criterion (exclusion): "No anticoagulant use" → checks for anticoagulant class → patient has warfarin → FAIL

4. Inclusion vs. Exclusion Logic

Criteria are split into inclusion (patient must meet) and exclusion (patient must NOT meet).

Criterion TypePatient Matches CriterionResult
InclusionYesPASS
InclusionNoFAIL
ExclusionYes (patient has excluded condition)FAIL
ExclusionNo (patient doesn't have it)PASS

For exclusion criteria, the pass/fail is inverted: if the evaluator finds the patient matches an exclusion condition, the final result is FAIL (patient is excluded from the trial).

5. Score & Eligibility Calculation

Match Score

Score = (Passed Criteria / Evaluated Criteria) × 100% "Evaluated" = criteria where we got a definitive PASS or FAIL. Unknown/unevaluable criteria are excluded from both numerator and denominator.

Eligibility Determination

Eligible = (Failed === 0) AND (Passed > 0) A patient is "potentially eligible" only if they failed zero evaluated criteria AND passed at least one.
Trial has 23 criteria: 5 evaluable (all pass), 18 unevaluable
→ Score = 5/5 = 100%
→ Failed = 0, Passed = 5 → POTENTIALLY ELIGIBLE
Important caveat: This approach is optimistic. A 100% score with only 5 of 23 criteria evaluated does not mean the patient truly qualifies — it means they passed everything we could check. The 18 unknown criteria (consent, clinical notes, etc.) could contain disqualifying conditions we cannot assess from structured data alone.

7. Known Limitations

  • Heuristic, not clinical: Rules are regex-based pattern matching, not validated clinical decision logic. They can miss unusual phrasing or complex criteria.
  • No clinical notes: MIMIC-IV demo excludes free-text notes, so S2 criteria (history, smoking, functional status) cannot be evaluated.
  • Simple negation: Negation detection uses keyword matching ("not", "no", "without"). Complex sentences like "patients with diabetes who do NOT have renal failure" may be misclassified.
  • Single lab value: Only the most recent lab value per patient is used. Criteria requiring trends, baselines, or values at specific timepoints cannot be assessed.
  • Incomplete drug lists: Drug name and class mappings cover common cardiovascular medications but are not exhaustive.
  • No vital signs: Blood pressure, heart rate, and other vitals from ICU chartevents are not currently included in the preprocessed patient data.
  • No compound criteria: Criteria with multiple conditions joined by "and"/"or" are evaluated as a single unit — only the first matching sub-condition is checked.
  • Optimistic eligibility: Patients can be marked "eligible" even when most criteria are unevaluable, as unknowns are excluded from the score denominator.
